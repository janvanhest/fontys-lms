# Leervragen - Studentgericht leerplatform

## Technology

Ik ben sterk in backend/full-stack ontwikkeling (Laravel, React, .NET, Docker). Maar deze challenge duwt me naar technologie die ik nog niet beheers:

- **Canvas API / LTI-integratie** - ik heb nog nooit tegen een LMS API gebouwd. Hoe werkt de Canvas REST API? Hoe implementeer ik een LTI-tool? Wat zijn de beperkingen?
- **AI-integratie in een educatieve context** - niet zomaar een chatbot, maar AI die competentiemapping doet of content ontsluit. Hoe maak ik dat betrouwbaar genoeg voor onderwijs?
- **UX/prototyping** - paper prototyping en gebruikerstesten zijn nieuw voor mij. Ik heb eerder vooral gebouwd, niet gevalideerd met gebruikers.

**Leervragen:**

- Hoe integreer ik een externe applicatie met Canvas via de API en LTI?
- Welke technologiestack is geschikt voor een tool die naast Canvas functioneert?
- Hoe zet ik AI in om content op alternatieve manieren aan te bieden?

## Context

De onderwijscontext is nieuw voor mij als ontwikkeldomein. Ik ben zelf student, maar ik heb nog nooit software ontworpen vanuit onderwijskundige principes.

- **Vraaggestuurd onderwijs** - wat betekent dit concreet voor systeemontwerp? Hoe verschilt dat van klassiek onderwijs?
- **Nudging in onderwijs** - hoe stimuleer je gewenst gedrag zonder te dwingen? Dat is geen technisch probleem maar een ontwerpprobleem.
- **Verschillende gebruikersgroepen** - dagstudenten opereren fundamenteel anders dan avondstudenten. Hoe ontwerp ik voor beide?
- **Gebruikersonderzoek** - ik heb weinig ervaring met het ophalen van behoeften bij gebruikers. Een enquete als breed startpunt en paper prototypes testen met een paar mensen lijkt me haalbaarder dan klassieke interviews.

**Leervragen:**

- Wat zijn de behoeften van studenten, coaches en docenten binnen vraaggestuurd onderwijs?
- Hoe vertaal ik onderwijskundige principes (personalisatie, zelfhulpzaamheid, nudging) naar concrete systeemfunctionaliteit?
- Hoe verschilt het gebruik van een LMS tussen dag- en avondstudenten?
- Hoe haal ik op een effectieve manier behoeften op bij gebruikers zonder te sturen op het antwoord?

## Doel

Ik wil Software op niveau 3 aantonen. Dat betekent: werken in een complexe, ongestructureerde context met bestaande systemen. Daar zitten leervragen aan vast die over mijn aanpak gaan.

- **Ideation als methode** - ik ben gewend om vrij snel te bouwen. Eric vraagt juist om eerst breed te verkennen, te scoren, en dan pas te kiezen. Dat is een andere manier van werken voor mij.
- **Stakeholdermanagement** - ik werk met Eric, maar ook met medestudenten, coaches, docenten, en dagschoolstudenten als testgroep. Hoe manage ik dat?
- **Ontwerpen voor overdracht** - toekomstige teams bouwen hierop voort. Hoe documenteer en architecteer ik zodat dat kan?

**Leervragen:**

- Hoe voer ik een ideation-fase uit en hoe scoor ik oplossingsrichtingen objectief?
- Hoe ontwerp ik een systeem dat overdraagbaar is naar toekomstige teams?
- Hoe onderbouw ik architectuurkeuzes in een context waar de requirements nog niet vaststaan?
